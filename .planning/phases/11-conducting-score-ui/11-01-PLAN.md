# PLAN 11-01 — Persist Layer + Conducting Score UI

> **ТЗ (self-execute, owner-decision 2026-07-24 "делай что логично, оптимально и рационально").**
> Wave 11: Orchestra получает визуальный Conducting Score на РЕАЛЬНЫХ данных LLM.
> Закрывает 3 долга одной фазой: D-9-6 (persist responses), D-8b-3 (confidence gauges), D-8b-5 (full score).
>
> **Ключевая находка разведки:** данные текут через существующий `GET /sessions/:id` —
> НЕТ НУЖДЫ в новом endpoint. Rounds встроены в `Session.rounds` (Prisma Json column).
> Добавление опциональных `responses?`/`consensus?` в Round → persist в adapter → данные
> автоматически появляются во фронтенде через существующий TanStack Query `useSession()`.

## 0. Контекст

### 0.1. Что не закрыто (3 долга)

**(1) D-9-6 — responses эфемерны.** `RoundOrchestratorGatingAdapter.evaluate` собирает
`roleResponses: RoleResponse[]` (loop lines 31-49), вызывает `consensus.run()` (line 52), получает
`ConsensusReport`, извлекает только `gatingVerdict` + gap counts (lines 58-62), и **всё остальное
выбрасывается**. `Round` interface (`packages/domain/src/session.ts:9-17`) не имеет поля `responses`.
Реальный LLM контент (claude-sonnet-4.5 critique, mimo-v2.5-pro code, gemini-2.5-flash research) —
consumed ConsensusEngine и уничтожается. Нет audit trail.

**(2) ConsensusReport discarded.** `ConsensusReport` (`consensus.ts:45-58`) несёт богатые данные:
`summary`, `agreedDecisions`, `disagreements` (Conflict[]), `openQuestions` (Question[]), `risks` (Risk[]),
`confidence` (DecisionConfidence — 6 метрик), `gatingVerdict`. Всё это считается детерминированно
(`confidence-calculator.ts` — claim coverage + agreement formulas, 0-100 scale) и **discarded**.

**(3) UI Canon не реализован.** `docs/UI Canon.md` описывает Conducting Score (staves=роли, measures=rounds,
chords=решения) + Confidence gauges (5 метрик как bars). Phase 8b (PASS) построил только CRUD + conduct
controls — §1 Score и §2 gauges явно отложены (D-8b-3, D-8b-5). Сейчас UI показывает round cards
(phase/status badge), НЕ real content, НЕ confidence.

### 0.2. Что делает Phase 11-01

**Phase A (backend persist) + Phase B (frontend UI), данные текут через существующий endpoint.**

### 0.3. Архитектурное решение: persist в adapter, без нового endpoint

**Выбор:** Option A из разведки — persist инкапсулирован в `RoundOrchestratorGatingAdapter`
(adapter имеет `report`, `roleResponses`, `round`, `session`, `this.store` — всё в scope).
0 изменений в `GatingResult`/`GatingPort`/`GsdEngine`.

**Альтернатива (отвергнута):** widen `GatingResult` чтобы нести `report?` — требует editing
`packages/gsd-engine/src/types.ts`, coupling engine package к ConsensusReport, выше blast radius.

**Frontend:** данные текут через `GET /sessions/:id` → `Session.rounds[].responses`/`.consensus`
(Prisma Json column round-trips automatically). 0 новых endpoints, 0 новых hooks.

### 0.4. Что НЕ меняется (anti-conflict)

- **`packages/gsd-engine/`, `packages/consensus-engine/`, `packages/role-router/`, `packages/providers/`,
  `packages/context-service/`, `packages/knowledge-graph/`, `packages/prompt-registry/`:** 0 изменений.
- **`packages/domain/src/`:** правка ТОЛЬКО `session.ts` (+2 опциональных поля в Round, +RoundResponse тип).
  `consensus.ts`, `agent.ts`, `decision.ts`, `events.ts` — 0.
- **`apps/api/src/`:** правка ТОЛЬКО `gsd/round-orchestrator-gating.adapter.ts` (persist before return).
  sessions controller, events controller, prisma schema, prisma-session-store — 0.
- **`apps/web/src/`:** 3 новых компонента + 2 modified (round-list, session-detail). Существующие
  conduct-controls, hooks, ui primitives — 0.
- **`docker-compose.yml`, Prisma schema, `role-manifests/`, `prompts/`, `docs/`:** 0 изменений.

## 1. Phase A — Backend persist layer

### 1.1. Domain: расширить Round (`packages/domain/src/session.ts`)

```typescript
// Импорт ConsensusReport (уже в этом пакете, consensus.ts:45)
import type { ConsensusReport } from './consensus.js';
import type { RoleRef } from './decision.js';
import type { Response } from './agent.js';

/** Per-role LLM response, persisted на Round для audit trail (D-9-6). */
export interface RoundResponse {
  role: RoleRef;
  response: Response;
}

export interface Round {
  id: RoundId;
  sessionId: SessionId;
  number: number;
  phase: GSDPhase;
  status: RoundStatus;
  startedAt: ISO8601;
  completedAt?: ISO8601;
  responses?: RoundResponse[];   // ← НОВОЕ: per-role LLM responses (claude/mimo/gemini контент)
  consensus?: ConsensusReport;   // ← НОВОЕ: full consensus report с confidence метриками
}
```

**Backward-compat:** опциональные поля (`?`). Старые rounds в БД (без этих полей) десериализуются
как `undefined` (PrismaSessionStore cast `as unknown as Round[]`).

### 1.2. Persist в adapter (`apps/api/src/gsd/round-orchestrator-gating.adapter.ts:52-62`)

```typescript
// После line 56 (const report = await this.consensus.run(...)), ДО return (line 58):

// Persist responses + consensus на round (D-9-6, D-8b-3, D-8b-5)
round.responses = roleResponses.map((rr) => ({ role: rr.role, response: rr.response }));
round.consensus = report;
round.status = 'completed';
round.completedAt = new Date().toISOString();
await this.store.update(session);
```

**Зачем `.map`:** `RoleResponse` из consensus-engine имеет тип `{ role; response }` — идентичен `RoundResponse`
по форме, но TS может требовать явный маппинг (или structural typing пропустит as-is). Если structural
compatibility работает — маппинг не нужен, прямой assign.

## 2. Phase B — Frontend Conducting Score UI

Стек: shadcn/ui + Tailwind + TanStack Query. NO new chart lib — gauges = Tailwind divs (width %).

### 2.1. ConfidenceGauges (`apps/web/src/components/confidence-gauges.tsx`)

UI Canon §2. 5 gauge-баров из `DecisionConfidence`:
- `architecture`, `implementation`, `researchCoverage`, `riskCoverage`, `testCoverage`.
- **НЕ `overall`** — Canon §9.2 запрещает сводный %.
- Каждый bar: Tailwind `div` с `style={{ width: '${value}%' }}`, `bg-primary` fill + `bg-muted` track.
- Цвет: green (value ≥ threshold), yellow (Δ ≤ 5% от threshold), red (value < threshold).
- Thresholds hardcoded (mirror `gating-thresholds.ts`): Goal→arch 70, Specification→research 75,
  Architecture→arch 85, Implementation→impl 80, Review→risk 70, Consensus→overall 80.
  Discover/Iteration → undefined (no threshold, all green).
- Labels на русском: «Архитектура», «Реализация», «Исследования», «Риски», «Тесты».
- Tooltip: значение + threshold (если есть).

### 2.2. ScoreStaves (`apps/web/src/components/score-staves.tsx`)

UI Canon §1. Горизонтальные дорожки ролей с РЕАЛЬНЫМ контентом:
- Каждый stave: `role.displayName` + `response.content` (обрезанный ~300 chars, expandable).
- НЕ chat-bubble (Canon §9.1) — дорожка в строку, структурированная.
- Role label слева, контент справа (как партитура: дорожка + ноты).
- Expand/collapse: клик разворачивает полный контент.
- Markdown-lite: basic formatting (headers `#`, bold `**`) через простую обработку или `<pre>`.

### 2.3. ConsensusPanel (`apps/web/src/components/consensus-panel.tsx`)

UI Canon §3. Из `round.consensus`:
- `summary` — текст.
- `gatingVerdict` — banner (green PASS / red FAIL).
- `disagreements` (Conflict[]) — каждый: topic + positions (role: claim).
- `openQuestions` (Question[]) — каждый: text + askedBy.
- `risks` (Risk[]) — каждый: description + severity badge + mitigation.

### 2.4. Расширение round-list.tsx + session-detail.tsx

**round-list.tsx:** каждый round card теперь рендерит (если данные есть):
```tsx
{round.consensus && <ConfidenceGauges confidence={round.consensus.confidence} phase={round.phase} />}
{round.responses && round.responses.length > 0 && <ScoreStaves responses={round.responses} />}
{round.consensus && <ConsensusPanel consensus={round.consensus} />}
```

**session-detail.tsx:** ConfidenceGauges последнего completed round — в шапку (Canon layout: gauges сверху).
Только если есть `session.rounds` с `consensus`.

## 5. Design notes

1. **Данные через существующий endpoint.** Round — embedded в Session (Prisma Json). Новые optional fields
   flow автоматически через `GET /sessions/:id` → TanStack Query. 0 новых endpoints/hooks.
2. **Persist в adapter (Option A).** Adapter имеет всё в scope. 0 изменений в GatingResult/GsdEngine.
3. **Backward-compat.** Optional fields (`?`). Старые rounds → undefined. PrismaSessionStore cast работает.
4. **NO chart lib.** Gauges = Tailwind divs (width %). Минимум зависимостей.
5. **Canon compliance.** §9.1 (no chat-bubbles), §9.2 (no overall %), §2 (5 gauges), §1 (staves).
6. **Thresholds hardcoded во frontend.** Mirror gating-thresholds.ts. Если thresholds станут pluggable (D-E4) —
   frontend обновится. Сейчас MVP — hardcoded OK.
